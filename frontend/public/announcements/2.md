# Trader Bot의 권한과 자동 역할 시스템 안내

안녕하세요! Trader Bot 개발팀입니다.

Trader Bot을 믿고 여러분의 소중한 디스코드 서버에 초대해 주셔서 진심으로 감사드립니다!

처음 서비스를 이용하시면서 _"디스코드 권한과 어떻게 연동되지?"_, _"누가 경매나 프리셋을 관리할 수 있지?"_ 하고 궁금해하시는 분들이 많습니다.
원활하고 안전한 내전 운영을 위해, Trader Bot 내부의 **[역할(Role)] 시스템과 자동 연동(동작) 방식**을 쉽고 명확하게 안내해 드립니다!

---

## 1. 어떻게 역할이 배정되나요? (자동 연동 방식)

Trader Bot은 웹에서 번거롭게 일일이 권한을 세팅하지 않아도, 디스코드 서버의 권한을 기반으로 아래와 같이 역할을 자동 부여합니다.

- **소유자(Owner):** 디스코드 서버를 소유한 서버장에게 자동으로 동기화되어 관리됩니다.
- **관리자(Admin):** 디스코드 내에서 `서버 관리자` 권한을 가진 멤버가 웹에 로그인하면 자동으로 부여됩니다.
- **열람자(Viewer):** 디스코드 서버 관리자 권한이 없는 일반 서버 멤버에게 기본적으로 가입 시 자동 부여되는 역할입니다.
- **권한 강등 처리:** 만약 디스코드 서버 내에서 관리자 권한을 잃게 되면, 웹에서도 기존에 수동으로 부여받은 '편집자' 역할이 없는 한 즉시 **열람자**로 자동 강등되어 내전 데이터가 안전하게 보호됩니다.

---

## 2. 역할별로 어떤 작업을 할 수 있나요? (권한 수준)

각 역할이 가지는 서비스 웹(웹사이트) 내의 권한 범위는 다음과 같습니다.

- **소유자 및 관리자:** 프리셋(템플릿)의 최초 생성부터 수정, 삭제까지 환경 설정의 **모든 관리 권한**을 행사합니다. 웹 내에서 다른 멤버에게 역할을 부여할 수도 있습니다. (단, 소유자 권한 자체는 타인에게 부여할 수 없습니다.)
- **편집자(Editor):** 프리셋이나 웹 설정 자체는 건드릴 수 없지만, 서버 내 멤버 정보나 프리셋에 등록된 **멤버의 스펙 및 포지션 수정을 전담**할 수 있는 실무 관리자 역할입니다.
- **열람자(Viewer):** 데이터의 수정이나 생성이나 삭제 권한은 없지만, 설정된 프리셋과 경매 현황 등을 전체적으로 **조회하고 확인**할 수 있는 일반 참가자 역할입니다.

더 디테일한 권한 제어나 헷갈리는 부분이 있다면, 주저하지 말고 언제든 공식 디스코드 서버를 통해 개발팀에 문의해 주세요!
