# v0.4.0

안녕하십니까. Trader Bot 운영팀입니다.

v0.4.0 버전은 **프리셋 복제 파이프라인 도입 및 목록 탐색 편의 기반 구축**을 통해 반복적인 운영 세팅 비용을 절감하고, 경매 처리 구조의 안정성과 인프라 대응력을 높이는 것에 목적이 있습니다.

## 추가사항

- **프리셋 복제:** 기존 프리셋의 설정·티어·포지션 구성을 그대로 복제하여 신규 프리셋 생성 시 초기 세팅 소요를 대폭 줄일 수 있습니다. 길드 간 복제도 지원합니다.
- **멤버 검색:** 멤버 목록 페이지에서 이름·별칭 기반 실시간 검색을 지원합니다.
- **프리셋 멤버 검색:** 프리셋 멤버 편집기 내 할당된 멤버 패널과 후보 멤버 패널 각각에 독립적인 검색 입력을 제공합니다.

## 개선사항

- **경매 상태 관리:** 경매 상태 및 입찰 처리 로직을 Redis 기반 구조로 전면 이관했습니다. 서비스가 재시작되더라도 진행 중인 경매 상태를 자동으로 복구합니다. 경매 이벤트는 Redis Pub/Sub 채널로 전달되고, 핵심 연산은 Lua 스크립트 기반 원자 실행으로 데이터 일관성을 보장합니다.
- **멤버 목록 페이지네이션:** 전체 일괄 로드 방식에서 커서(Cursor) 기반 무한 스크롤 방식으로 전환했습니다. 멤버 수가 증가해도 초기 로딩 부하 없이 탐색할 수 있습니다.
- **프리셋 멤버 목록 페이지네이션:** 프리셋 멤버 편집기 및 프리셋 멤버 그리드에도 동일한 커서 기반 페이지네이션을 적용했습니다.
- **경매 채널 지정:** 디스코드 슬래시 명령어(`/set_invite_channel`)를 통해 경매 초대 알림을 수신할 채널을 지정하거나 해제할 수 있습니다.
- **디스코드 봇 멤버 동기화:** 길드 멤버 동기화 시 발생하던 건별 반복 쿼리를 청크 단위 Bulk Upsert 방식으로 전환하여 대규모 길드 환경에서의 처리 시간을 단축했습니다.
- **FK 조인 구간 인덱스:** 멤버 조회 시 빈번히 사용되는 `guild_id`, `user_id` 외래 키 컬럼에 인덱스를 추가하여 JOIN 및 조건 필터링 구간의 쿼리 비용을 낮췄습니다.
- **입찰 UX:** 입찰 가능 범위(최솟값·최댓값·포인트 단위)를 입력 필드에 실시간으로 반영하고, 유효하지 않은 금액 입력 시 버튼을 비활성화합니다. 입찰 오류는 모달(시스템 오류)과 인라인(입찰 조건 오류)으로 분리하여 상황별 피드백을 제공합니다.

## 참고사항

- 프리셋 복제 시 멤버 배정 데이터는 복제 대상에 미포함. 설정·티어·포지션 구조만 복사
- 검색 기능은 입력 후 300ms 지연(Debounce) 이후 서버 요청 전송. 별칭 및 이름 필드에 대해 부분 일치(ILIKE) 방식으로 조회
- 커서 기반 페이지네이션은 목록 하단 도달 시 다음 항목을 자동으로 불러오는 무한 스크롤 방식으로 동작
- 경매 상태는 Redis에 TTL과 함께 저장. 서비스 재시작 시 TTL 미만료 진행 중 경매 자동 감지 및 복구. 완료된 경매는 복구 대상 제외
- Bulk Upsert는 일정 크기 단위로 분할(Chunk)하여 처리. 수천 명 규모 대형 길드에서도 단일 트랜잭션 크기 제어 가능
- 입찰 금액 유효성은 포인트 단위(pointScale) 배수 여부, 최솟값 초과, 보유 포인트 범위 이내를 복합 검사
