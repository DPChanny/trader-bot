# v0.3.0

안녕하십니까. Trader Bot 운영팀입니다.

v0.3.0 버전은 **운영 안정성 확보 및 인프라 아키텍처 개편**을 통해 서비스의 배포 프로세스 표준화를 확립하는 것에 목적이 있습니다.

## 추가사항

- **배포 프로세스:** dev, beta, prod 3단계 배포 플로우를 확정하고 선택적 베타 배포(beta.trader-bot.org) 기준을 수립하여 프로덕션 반영 전 사전 검증 단계를 체계화했습니다.
- **인프라 코드화(IaC):** 인프라 변경 이력을 코드 기반으로 추적하고 장애 발생 시 신속하게 원상 복구할 수 있는 구조적 기반을 확보했습니다.
- **패치 노트 체계:** 버전 단위 변경 이력을 일관되게 제공할 수 있는 패치 노트 게시 시스템을 구축했습니다.
- **공지 게시 체계:** 운영 정책 및 주요 변경 사항을 신속하게 전달할 수 있는 공지 시스템을 마련했습니다.

## 개선사항

- **인프라 구성:** AWS 인프라를 전면 개편하여 서비스 토폴로지의 일관성과 확장성을 높였습니다.
- **배포 자동화:** GitHub Actions 기반 워크플로를 구축하여 파이프라인 일관성을 확보하고 배포 처리 속도를 개선했습니다.
- **릴리즈 기록 체계:** 운영 관점의 이슈 대응 및 변경 추적 효율을 향상시켰습니다.

## 참고사항

- dev, beta, prod 환경별 반영 순서 고정 및 beta 환경 사전 검증 후 상위 환경 승격 기준 적용
- IaC 기반 인프라 변경 절차의 사전 검토·적용·사후 검증 단계 표준화 및 배포 중단(Downtime) 가능성 최소화
- 패치 노트와 공지사항의 릴리즈 시점 동시 반영 원칙 적용
