# v0.3.0

안녕하십니까. Trader Bot 운영팀입니다.

v0.3.0 버전은 **운영 안정성 확보 및 인프라 아키텍처 개편**을 통해 서비스의 배포 프로세스 표준화를 확립하는 것에 목적이 있습니다.

## 추가사항

- **배포 프로세스 확립:** dev, beta, prod 3단계 배포 플로우 확정 및 선택적 베타 배포(beta.trader-bot.org) 기준 수립
- **인프라 코드화(IaC):** 인프라 변경 이력의 코드 기반 추적 및 장애 대응 기반 확보
- **패치 노트 체계 신설:** 버전 단위 변경 이력을 일관되게 제공하는 시스템 구축
- **공지 게시 체계 정비:** 운영 정책 및 주요 변경 사항의 신속한 전달 체계 마련

## 개선사항

- **인프라 구성 개편:** AWS 인프라 구성을 전면 개편하여 서비스 토폴로지의 일관성 및 확장성 증대
- **배포 자동화:** GitHub Actions 기반 워크플로 구축으로 파이프라인 일관성 문제 해결 및 처리 속도 개선
- **릴리즈 기록 체계 정비:** 운영 관점의 이슈 대응 및 변경 추적 효율 향상

## 참고사항

- dev, beta, prod 환경별 반영 순서를 고정하고, beta 환경에서의 사전 검증 이후 상위 환경으로 승격하는 엄격한 기준을 적용했습니다.
- IaC 기반 인프라 변경 절차를 사전 검토, 적용, 사후 검증 단계로 표준화하여 배포 중단(Downtime) 가능성을 최소화했습니다.
- 패치 노트와 공지사항을 릴리즈 시점에 동시 반영하는 원칙으로 조정하여 사용자 커뮤니케이션 공백을 줄였습니다.
