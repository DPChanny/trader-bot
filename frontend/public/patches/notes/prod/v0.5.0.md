# v0.5.0

안녕하십니까. Trader Bot 운영팀입니다.

v0.5.0 버전은 **경매 이벤트 프로토콜 정비 및 운영 로그 가시성 강화**를 중심으로, 서비스 내부 안정성과 운영 추적 품질을 높이는 데 목적이 있습니다. 또한 이후 릴리즈에서 제공될 구독 기반 결제 기능의 백엔드 기반을 사전 구축했습니다.

## 추가사항

- **결제 API 기반 구축:** 구독 기반 결제 서비스 연동을 위한 백엔드 API를 추가했습니다. 결제 화면 UI는 v0.5.1에서 제공됩니다.

## 개선사항

- **경매 이벤트 타입 체계 정비:** `AuctionClientEventType`, `AuctionServerEventType`, `AuctionPublishType`, `AuctionRequestType` 열거값을 연속된 0~n 범위로 정규화하고, `AuctionServerEventType`이 `AuctionPublishType`과 독립적인 값 범위를 가지도록 상속 구조를 재편했습니다.
- **운영 로그 가시성 강화:** 이벤트 로그의 `detail` 필드를 `request_type` / `publish_type` / `server_event_type`으로 명확하게 분리하고, 열거형 값을 정수 대신 이름 문자열로 직렬화하여 로그만으로 이벤트 유형을 즉시 파악할 수 있도록 개선했습니다.
- **CloudWatch 에이전트 재시작 누락 수정:** 배포 시 CloudWatch 에이전트가 새 로그 설정을 반영하지 않던 문제를 수정했습니다.

## 참고사항

- 경매 이벤트 타입 값이 변경됨에 따라 프론트엔드와 백엔드를 동시에 배포해야 합니다. 버전 혼용 시 경매 이벤트가 정상적으로 처리되지 않을 수 있습니다.
- 결제 UI는 v0.5.1, 구독 기반 기능 제한은 v0.5.2에서 순차적으로 제공됩니다.
